Software Modules

Three main areas that define
the Spectra software experience.

Spectra is structured around a practical software flow. The editor handles direct script work, the built-in hub improves discovery, and the settings panel gives the user centralized control over behavior and startup actions.

Module 01

The Editor is the main working environment.

This is the core screen of the software. It provides a focused script editing area, tab support, direct execution controls, output monitoring, and access to local files. The layout is designed so the most common actions stay visible and easy to reach.

Tabbed editing workflow for handling multiple files in a more organized way.
Integrated execute, clear, open, save, and attach actions in the same workspace.
Built-in explorer panel for faster access to local script files and saved content.
Module 02

The Script Hub expands the client beyond a plain editor.

Spectra is not limited to manual editing alone. The hub page gives the software a broader, more complete feel by providing a dedicated space for browsing script content, preview cards, and quick interaction without leaving the main product environment.

Search-based script discovery integrated directly into the software layout.
Card-based presentation that feels more modern and visually structured.
A stronger ecosystem feel compared to a client that only offers raw execution tools.
Module 03

The Settings panel acts as the software control center.

The settings area is where system behavior becomes configurable. Startup actions, language, auto update, auto attach, automatic execution, and performance-related options are presented in a cleaner interface that is easy to understand at a glance.

Quick access to client behavior controls without cluttering the main editor.
Separate categories for system, Windows, and editor preferences.
A cleaner configuration surface that helps the software feel complete and polished.
